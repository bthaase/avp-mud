Chapter 1 - Introduction to Avp
Section 1.1 - Codebase
AVP Building is always online. It uses a set of commands that are contained within the MUD to create new areas and rooms, that are interconnected to other areas. Since the new area is created and saved online, there is no need to "upload" or "install" areas, other than a simple Implementor command.

Section 1.2 - Whats 'Prototype' mean?
Now, when you  first get your vnums (VIRTUAL NUMBERS, which are the numbers that hold your new areas information, like rooms, mobiles, objects, that kind of stuff), the area is created with your name and '.are' at the end of it. So if I assigned myself some vnums, my new area would be called "Ghost.are" and the area would loadup everytime I logged in, if its not already loaded.

Since it's a new area, yet to be finished, it is a "prototype" area, meaning everything in it is new and isn't necessarily going into the finished product. No area should be installed until it is marked "Done" by the Head Builder or Game Administrator. Likely me.

Section 1.3 - Why build when I can play?
Ok, most old builders will tell you that building is no fun. This is somewhat true... Building takes a lot of effort and drains out a lot of your time. You'll usually have something fail on you and it will drive you nuts. But that's ok. Most things in life screw up once in a while and it takes a strong person to get back in there and start rolling.

What old builders don't tell you is the satisfaction you get when you hear some player yell how great the new Airbase is or how much better this arena is than from the one they just played. And the little hidden secrets you put in your area and the programs you install make everything worthwile. Exploring is for the kiddies. Creating is what the true of heart do. So get out there and start building!

Thanks to Travis Shofner (Isaac) for the SWR Building Guide.

Chapter 2 - Designing your Area
Section 2.1 - Introduction
 To begin with, theres a few commands you should know. First off, HELP is an important one. As of writing this, not all of the help files are done, but eventually we'll get them all up. If your feeling lost while building, or are confused about how something works, try using HELP to look it up. Also, theres always the WIZHELP command, which will list all of the immortal commands at your disposal. 

Another couple of commands you might find interesting:

SAVEAREA will save your area to disk, so your work won't be lost if the mud crashes or reboots. Use this command a lot, and make sure to do it when you log off also. You can use this command a lot without putting much strain on the server, so make sure to do it a lot. 

ZONES will list all of the areas on the mud, and there vnum ranges. Your area will be on the list near the top, as <Your Name>.are - If the zone has a * by it, then it iss loaded. If your area isn't flagged as loaded, you can use LOADAREA to load it.

Section 2.2 -Important Notes
 Theres a couple of important factors when designing your area.  We'll assume you know how many vnums you'll have by now. Firstly, when designing your area, DRAW IT OUT ON GRAPH PAPER FIRST. Otherwise, if you just do it on the fly you risk ruining the coordinate-mapping system - automated systems are only capable of so much.

 In addition to mapping it out  on paper first, make sure you pay attention to the layout carefully. If you do it wrong, You *WILL* have to fix it, or if theres too much to fix, you have to start over. Nobody wants that. What kind of screwups are you avoiding? First off, DONT OVERLAP ROOMS ON THE MAP. Also, DONT LEAVE GAPS. If you have four interconnecting rooms in a square, and you go N | E | S then W, you should be back at the beginning. If your NOT, then you screwd up. The nature of the coordinate system is that it requires NO work to setup on your area, but that if your layout is wrong, it'll goto hell really fast.

 Also, EACH RACE NEEDS A SPAWN POINT. These must be flagged DEPLOY_M (For Marines), DEPLOY_A (For Aliens) and DEPLOY_P (For Predators). When a player uses the DEPLOY command, it will take them to one of the flagged rooms for their race. 

In Summary:
DRAW IT OUT ON GRAPH PAPER FIRST!
DONT OVERLAP ROOMS ON THE MAP.
DONT LEAVE GAPS ON THE MAP.
 EACH RACE NEEDS ATLEAST ONE SPAWN POINT.

Chapter 7 - Advanced Scripts Reference
This section contains prewritten and some slightly more advanced mobprog scripts that you can use in your areas. The script listing is included, as well as a summery of how it works and how to install it.

Section 7.1 - Circuit Breakers
  

First Script
mpechoat $n You throw the break to the ON position. 
mpechoaround $n $n throws the breaker to the ON position. 
mprat (1) (2) mprmflag -1 1 dark
mprat (1) (2) mpecho The overhead lights flare to life.
 

Second Script
mpechoat $n You throw the break to the OFF position. 
mpechoaround $n $n throws the breaker to the OFF position. 
mprat (1) (2) mprmflag -1 0 dark
mprat (1) (2) mpecho The room suddenly goes dark.

Circuit breakers are pretty straight-forward. Create an object, and set the USEON flag. This allows players to USE the object. Next, use OPEDIT to add a USEON program. (opedit <vnum> add useon 100) and paste the first block of code in there. Replace (1) and (2) with the first and last vnum affected. Use /S to save the program, and then add a USEOFF program. (opedit <vnum> add useoff 100) and paste the second block of code in there. Replace (1) and (2) with the first and last vnum affected. Use /S to save the program.

How does it work? First off, the MPECHOAT command sends a message to the player who used the breaker switch, to inform them of the action - By default, the game assumes YOU will inform the player, so it doesn't send any of its own messages. Next, MPECHOAROUND does the same thing except it informs everyone in the room with you. The next two lines are the most important. MPRAT runs through a range of rooms, executing a command in each one - It uses MPRMFLAG to set or clear or the dark flag in each of the rooms, and then MPECHO to let everyone know the lights just went out. Pretty simple.

Section 7.2 - Camera Consoles
  

Console Script
mpechoat $n You glance at the camera console. 
mptransfer $n (1)
mpat (1) mpforce $n glance 
mpat (1) mptransfer $n (2)

Camera consoles are actually pretty easy. Create an object, and set the USEAUTO flag. This allows players to USE the object to trip the USEON and USEOFF programs at once. Next, use OPEDIT to add a USEON or USEOFF program. (opedit <vnum> add useon 100) and paste the block of code in there. Replace (1) and (2) with the vnum of the room to 'spy' on, and the vnum of the room the camera console is in. Use /S to save the program.

How does it work? First off, the MPECHOAT command sends a message to the player who used the camera console, to inform them of the action - By default, the game assumes YOU will inform the player, so it doesn't send any of its own messages. Next, MPTRANSFER actually moves you to the room thats being 'spied' on. MPAT allows the script to run at the camera location, otherwise it can't always find the player. It uses MPFORCE to the player to glance at the room contents, and then  uses another MPTRANSFER to bring them back to the starting room.

Section 7.3 - Bypass Consoles

Bypass Console Script
if variable([1]) == 0
mea $n You call up the administration controls on the console.
mer $n $n carefully accesses the security console.
mpset [1] 1
mpecho &w&G[Notice]: Security System has been bypassed.
else
mea $n You call up the administration controls on the console.
mer $n $n carefully accesses the security console.
mpset [1] 0
mpecho &w&R[Warning]: Security System has been reactivated.
endif

Create an object, and set the USEAUTO flag. This allows players to USE the object to trip the USEON and USEOFF programs at once. Also, set the MARINE flag so that only Marines can use this console - Predators and aliens don't know very much about computers. Next, use OPEDIT to add a USEON or USEOFF program. (opedit <vnum> add useon 100) and paste the block of code in there. Replace [1] with the name of the variable to use, personally I use SecurityBypass. Use /S to save the program.

How does it work? This one is a bit more tricky. The 'if variable' check allows you to see what the variable is set to. (Default is 0) MEA and MER are just short names for MPECHOAT and MPECHORAOUND, respectively. MPSET allows you to set the variable, so that you can check it anywhere in your area. When first run, the flag will be set to 1, and the target system is considered 'offline'. When run again, the flag is set back to 0 and the target system is considered to be back online.. This way if  you shutdown the only defense between you and the fourty aliens outside the compound, you can repair your mistake ;)

Appendix B - Room Flags
dark - Makes the room dark, and players can't see. Aliens however, can see in the dark.
nomob - Mobiles aren't allowed to enter this room. Bots are an exception to this rule.
indoors - This room is considered 'inside'. Weather won't affect these rooms, nor night time.
can_drive - Players can drive vehicles into this room. Only needed for indoor rooms, outdoors are automatically can_drive.
nodropall -  Playes cannot use the DROP ALL command here. Not very useful.
logspeech - All speech in this room is logged to file. Not very useful.
teleport - Room will teleport its contents somewhere else after a period of time. 
nofloor - This room has no floor. Anything in will fall into the room below it, assuming theres an open down exit.
nodrop - Players cannot use the DROP command in here. Good for litter prevention.
prototype - This room is still prototype. Dont clear these on your own, they'll clear when you install the area.
save_eq - Anything dropped in this room will be saved between crashes and copyovers. Do NOT put resets in these rooms.
underground - This room is considered to be underground. Right now, this is essentially the same as indoors.
fallcatch - If a player falls into this room, they will take no damage.
lit - This room is always lighted, even at night time. Good for outdoor rooms that might have floodlights or something.
deploy_from - The DEPLOY command can be used from here to reach the current arena. Not for use IN arenas.
hived - Aliens have hived this room, which causes the secondary description to show up.
mapstart - Used for the coordinate automapper. Only one of these is needed, and will be set by the administrator.
nohive - Related to hived, this flag prevents aliens from hiving the room and marines from dehiving. 
safe - No combat damage may be dealt to occupents of this room. 
deploy_p - Predators can deploy to this room using the DEPLOY command.
deploy_m - Marines can deploy to this room using the DEPLOY command.
deploy_a - Aliens can deploy to this room using the DEPLOY command.
vented_a - Once installed, AvP will link all of room flagged vented_a together using the coodinates. 
vented_b - Once installed, AvP will link all of room flagged vented_b together using the coodinates. 
vented_c - Once installed, AvP will link all of room flagged vented_c together using the coodinates. 
vented_d - Once installed, AvP will link all of room flagged vented_d together using the coodinates. 
rescue - Marines may return rescued hostages to this room for experience.
cp - Control Point. Races can capture these to gain team experience and an extra spawn point.

Appendix C - Object Flags
glow - Just for effect really, adds the (Glowing) flag to the start of the description.
hum - Same as glow, except it adds (Humming) to the start of the description.
invis - Invisible to the naked eye. At the momemnt, only immortals can see invisible. 
nodrop - Once picked up, players cannot drop the item.
noremove - Once equipped, players cannot remove the item.
inventory - Used with shopkeepers, gives the unlimited of the item to sell. (They just keep copying it)
hidden - Similar to invisible, but... umm. Actually, its a lot like invisible.
covering - Depreciated. I should probably remove this.
deathrot - Item is deleted when the person carrying dies.
burried - Item is considered 'burried' and can only be found by digging. Might be used with landmines or something.
prototype - This item is still prototype. Dont clear these on your own, they'll clear when you install the area.
nodual - Used for weapons, makes it so players can't dual wield it. (Like Shotguns and Assault Rifles and such)
marine - Item may only be used by marine characters.
predator - Item may only be used by predator characters.
alien - Item may only be used by alien characters.
respawn -Item will stay with its owner when they die. Otherwise, its placed in your backpack/corpse.
burstfire - Used for weapons, allows it to be set to burstfire mode.
autofire - Used for weapons, allows it to be set to automatic fire mode.
semiauto - Used for weapons, allows it to be set to semiautomatic fire mode.
singlefire - Used for weapons, allows it to be set to single fire mode.
useon - Item is currently 'on' and when used will be set to USEOFF and trip any USEOFF programs.
useoff - Item is currently 'off' and when used will be set to USEON and trip any USEON programs.
quiet - Used for weapons, creates a quiet echo when the weapon is fired. (About 1 room)
moderate - Used for weapons, creates a moderate echo when the weapon is fired. (About 2 rooms)
loud - Used for weapons, creates a loud echo when the weapon is fired. (About 3 rooms)
rangedonly - Used for weapons, makes it so you can only sniper and spray with the weapon - Can't be used in the same room.
indestructable - Immune to explosions, smashing, all that good stuff. Can still be purged.
nohold - You cannot hold anything while wielding this weapon. Sort of like nodual, but applies to all items.
alieninvis - Only aliens are able to see this item. Used for Alien Constructions.

Appendix E - Mobile Specials
SPEC_ENEMY_SCAN
Allows mobiles to scan up to PER/3 rooms away to spot enemies. Once spotted, they will be moved into attack mode and attempt to fight the enemy. If its an alien mobile, it will chase down the enemy. If its any mobile armed with a ranged weapon, it will attempt to snipe its target to death.

SPEC_ENEMY_GRENADE
Quite similar to spec_enemy_scan, except that it allows the mobile to use any grenade items in his possession. It'll automatically hold a grenade if it has a free hand, and when an enemy is spotted at range it'll arm for 5 seconds and lob the grenade to the target.

Appendix H - Exit Flags
isdoor - Pretty important. Flags an exit as a door (open by default) that can be opened and closed.
closed - Works with isdoor, causes the door to count as 'closed' so people can't go through it.
locked - Works with closed doors. The door cannot be opened while locked.
secret - Renders the exit invisible to the naked eye. It can only be detected with SEARCH.  Obsolete.
swim - Requires your to be able to swim in order to use this exit..  Obsolete.
pickproof - Makes the door completely unpickable.
fly - Requires you to be flying in order to use this exit.  Obsolete.
climb - Requires you to be able to climb in order to use this exit.  Obsolete.
dig - Requires you to dig out the exit before travelling in that direction.  Obsolete.
nopassdoor - You dont even wanna know.  Obsolete.
hidden - The exit is hidden, and wont show on the exit list in the room. However, it can still be used.
passage - Honestly? I couldn't figure this one out. I assume its pointless right now.
portal - Not used anymore.  Obsolete.
can_climb - Not used anymore.  Obsolete.
can_enter - Allows you to 'enter <keyword>' to use the exit. Not used very often.
can_leave - Allows you to 'leave <keyword>' to use the exit. Not used very often.
auto - If you set a keyword for this exit, you can use the exit by just typing the keyword.
searchable - Allows you to use SEARCH to strip the secret flag off this exit.  Obsolete.
bashed - Used to show that a door has been bashed open by a player.
bashproof - Prevents players from bashing the door in. Armored doors has the same affect.
nomob - Mobile characters cannot use this exit while wandering.
window - Predators and Marines cannot go through these exits, only shoot through. Aliens, however, can go through windows.
can_look - Allows you to use LOOK (direction) to see through the exit, similar to scan.
keypad - Indicates that instead of a key, the player must enter a code to lock and unlock this door.
barred - Used on WINDOW Exits, prevents aliens from travelling through it.
nospec - Players in spectator mode may not use this exit. Good for keeping them from scouting key areas.
blastopen - Grenade explosions are required to blast this exit open and make it passable.
blasted - Shows that this exit has already been blasted open, and that players may travel through it.
armored - Used to indicate that a door is armored. Armored doors are immune to grenade blasts and bashing.
novent - Alien vents dont travel through this exit, good for keeping them out of key areas.
